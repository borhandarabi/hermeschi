# گزارش خطای Conductor

## ۱. وظایف portable Conductor هرگز اجرا نمی‌شدند

- علامت: ماموریت‌های حالت portable به‌عنوان وظایف زمان‌بندی‌شدهٔ Hermes ایجاد می‌شدند، ولی وظایف در حالت `scheduled` باقی می‌ماندند و هرگز اجرا نمی‌شدند.
- راه‌حل: portable Conductor اکنون به‌جای مسیر وظایف مرده، از مسیر موجود `/api/send-stream` برای استریم نشست استفاده می‌کند.
- اعتبارسنجی: آزمون دودویی API حالت portable رویدادهای SSE با نام‌های `started`، `chunk` و `done` را بازگرداند و build با موفقیت گذر کرد.

## ۲. ماموریت مبتنی بر داشبورد در حال اجرا بود، ولی رابط کاربری `0 active` را نشان می‌داد

- علامت: صفحهٔ conductor یک ماموریت مبتنی بر داشبورد را راه‌اندازی کرد، ولی پنل فعالیت حتی زمانی که داشبورد نشست‌های زندهٔ ماموریت را نشان می‌داد، در `0 active` باقی می‌ماند.
- راه‌حل: فیلتر نشست conductor اکنون نشست‌های اخیر مرتبط با ماموریت را بر اساس کلید دقیق و همچنین متن/خلاصهٔ ماموریت تطبیق می‌دهد، نه فقط برچسب‌های `worker-*` / `conductor-*`.
- اعتبارسنجی: پس از بارگذاری مجدد صفحهٔ conductor، ماموریت `1 active` را نشان داد و کارت worker ظاهر شد.
