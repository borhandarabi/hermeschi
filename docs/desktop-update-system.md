# سیستم به‌روزرسانی دسکتاپ HermesChi

این شاخه، قرارداد به‌روزرسانی را معرفی می‌کند که بسته‌بندی DMG/EXE باید از آن استفاده کند.

## محصولات

Hermes دو محصول جداگانه با قابلیت به‌روزرسانی مستقل ارسال می‌کند:

۱. **HermesChi**: پوستهٔ رابط کاربری/سرور.
۲. **Hermes Agent**: زمان اجرا و دروازه/عامل محلی.

این دو نباید به‌عنوان دو remote در یک checkout یکسان git مدل شوند. به‌روزرسان فضای کار، فضای کار را به‌روزرسانی می‌کند. به‌روزرسان Agent، Agent نصب‌شده/باندل‌شده را به‌روزرسانی می‌کند.

## API

- `GET /api/update/status`
  - نسخهٔ فضای کار + Agent و وضعیت نصب/به‌روزرسانی را برمی‌گرداند.
- `POST /api/update/workspace`
  - به‌روزرسانی فضای کار را فقط در شرایط امن اعمال می‌کند.
- `POST /api/update/agent`
  - به‌روزرسانی Agent را فقط در شرایط امن اعمال می‌کند.

## انواع نصب

پیاده‌سازی فعلی موارد زیر را شناسایی می‌کند:

- `git`: checkout توسعه/سورس.
- `docker`: اجرا در کانتینر، به‌روزرسانی درون‌فرآیندی اعمال نمی‌شود.
- `desktop`: رزرو‌شده برای یکپارچه‌سازی auto-updater با DMG/EXE.
- `unknown`: به‌روزرسانی خودکار به‌صورت امن ممکن نیست.

## رفتار Git/توسعه

برای نصب‌های git:

- به‌روزرسانی‌های فضای کار از `origin/<branch>` استفاده می‌کنند و نیاز به checkout تمیز و fast-forwardable دارند.
- به‌روزرسانی‌های Agent از دستور `hermes update` خود Agent فراخوانی می‌کنند و نیاز به checkout تمیز Agent دارند.
- حالت‌های کثیف یا غیر fast-forward مسدود شده و به‌عنوان «نیازمند بازبینی» نمایش داده می‌شوند، نه به‌عنوان مسیر اصلی copy-command.

## رفتار دسکتاپ برای اتصال در گام بعدی

اپ بسته‌بندی‌شده باید `HERMESCHI_DESKTOP=1` را تنظیم کند و یک پل به‌روزرسان دسکتاپ ارائه دهد که:

۱. یک manifest به‌روزرسانی امضاشده یا GitHub Release را بررسی می‌کند.
۲. به‌روزرسانی اپ فضای کار را از طریق Electron auto-updater یا معادل آن دانلود می‌کند.
۳. payload مربوط به Hermes Agent باندل‌شده را جداگانه به‌روزرسانی می‌کند.
۴. پس از به‌روزرسانی، فضای کار + Agent را دوباره راه‌اندازی می‌کند.
۵. یادداشت‌های انتشار را برای نخستین صفحهٔ بعد از به‌روزرسانی ذخیره می‌کند.

رابط کاربری همین حالا وضعیت به‌روزرسانی در سطح محصول و یادداشت‌های انتشار را انتظار دارد، بنابراین پل دسکتاپ باید به همان قرارداد `/api/update/*` نگاشت شود.
